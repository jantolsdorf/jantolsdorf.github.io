GitHub Multiple Accounts via SSH

This setup allows using multiple GitHub accounts on the same machine without constantly switching credentials.

SSH Configuration

File: ~/.ssh/config

Host github.com-NAME_ONE
  HostName github.com
  User git
  AddKeysToAgent yes
  UseKeychain yes
  IdentityFile ~/.ssh/YOUR_KEY_FOR_ACCOUNT_ONE
  IdentitiesOnly yes

Host github.com-NAME_TWO
  HostName github.com
  User git
  AddKeysToAgent yes
  UseKeychain yes
  IdentityFile ~/.ssh/YOUR_KEY_FOR_ACCOUNT_TWO
  IdentitiesOnly yes

Each Host entry creates a local alias for GitHub. SSH uses the specified key whenever that alias is used.

Adding Keys to the SSH Agent

Add the SSH keys to the SSH agent.

ssh-add ~/.ssh/YOUR_KEY_FOR_ACCOUNT_ONE

ssh-add ~/.ssh/YOUR_KEY_FOR_ACCOUNT_TWO

Verify which keys are currently loaded:

ssh-add -l

Testing Authentication

Test whether the configuration is working correctly.

ssh -T git@github.com-NAME_ONE

ssh -T git@github.com-NAME_TWO

GitHub should respond with the username associated with the corresponding SSH key.

Repository Setup

When cloning repositories, use the correct host alias defined in the SSH config.

For account one:

git clone git@github.com-NAME_ONE:username/repository.git

For account two:

git clone git@github.com-NAME_TWO:username/repository.git

Existing repositories can be updated using: 

git remote set-url origin git@github.com-NAME_ONE:username/repository.git

Troubleshooting

Authorization Error

This is usually caused by SSH not having the correct key loaded.

1. Check which keys are loaded:

ssh-add -l

2. Add the missing key:

ssh-add ~/.ssh/YOUR_KEY

3. Test authentication again:

ssh -T git@github.com-NAME_ONE

See Exactly Which Key SSH Uses

ssh -Tvvv git@github.com-NAME_ONE

Look for output similar to:

Offering public key: ~/.ssh/YOUR_SSH_KEY

This shows which key SSH is attempting to use for authentication.

If a repository uses: [git@github.com](mailto:git@github.com):user/repository.git instead of one of your configured aliases, SSH will bypass your custom host configuration and may authenticate with the wrong GitHub account.
