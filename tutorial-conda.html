Conda + conda-forge + Jupyter + VS Code Setup

Why conda-forge (and why not default Anaconda)?

Conda itself is free, but package sources matter. The default Anaconda channel includes packages tied to commercial licensing terms in some environments.

conda-forge is a community-maintained package ecosystem that is:

  • Fully open-source
  • Widely used in scientific computing
  • More up-to-date than default channels
  • Safe for corporate and commercial use without licensing concerns

Recommended setup: Miniforge + conda-forge only

1. Install and Configure Miniforge (one time setup)

Note: You have to do this only once; if you already have Miniconda or Anaconda installed, you can skip this step.

Miniforge is a minimal Conda installer configured to use conda-forge by default.

Download via this GitHub repository: https://github.com/conda-forge/miniforge#install

After installation, verify:

conda --version

Initialize shell integration:

conda init

Check current channels:

conda config --show channels

Force conda-forge priority:

conda config --add channels conda-forge
conda config --set channel_priority strict

This ensures all packages come from conda-forge and avoids mixing ecosystems.

2. Create a Jupyter Notebook Environment

Create environment:

conda create -n YOUR_PROJECT_NAME python=3.12 jupyterlab ipykernel

Activate it:

conda activate YOUR_PROJECT_NAME

Optional kernel registration:

python -m ipykernel install --user --name YOUR_PROJECT_NAME --display-name "Python (notebook)"

Start Jupyter:

jupyter lab

3. Use Conda Environment in Codium (or VS Code)

Note: Codium is the open source version of VS Code. It works the same way as VS Code.

Install Codium/ VS Code extensions:

  • Python (Microsoft)
  • Jupyter (Microsoft)

Open project folder:

codium . (if Codium) or code . (if VS Code)

Select interpreter in Codium/ VS Code:

  • Open Codium or VS Code
  • Open Command Palette by pressing the following keys:
    • Ctrl + Shift + P (Windows, Linux)
    • Command + Shift + P (MacOS)
  • From dropdown, select Python: Select Interpreter
  • From dropdown, select Python x.xx.xx (YOUR_PROJECT_NAME) Conda

Verify inside terminal:

conda activate notebook
                                python --version

4. Export Environment for Reproducibility

Once your environment is stable, you can export it to a YAML file so it can be recreated exactly on another machine.

This ensures reproducibility by capturing all installed packages and versions.

Export only manually installed dependencies (recommended):

conda env export --from-history > environment.yml

This produces a clean, minimal file like:

name: notebook
channels:
  - conda-forge
dependencies:
  - python=3.12
  - numpy
  - pandas
  - jupyterlab

To recreate the environment elsewhere:

conda env create -f environment.yml

For stricter reproducibility (including exact versions), use: 

conda env export > environment-full.yml
